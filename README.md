# SonicDE Branding Guidelines

## Conditions of usage

1. The term "SonicDE logo" refers to the SonicDE logo and other official artwork/mark. It also includes the official color scheme used by the project.

2. The term "SonicDE library" refers to the SonicDE tooling, associated libraries and official projects in the SonicDE ecosystem.

3. Usage of the SonicDE logo must not give the impression or implication that the SonicDE project (or any contributor to the project) is sponsoring or endorsing any other project, service, product or organization.

4. Usage of the SonicDE logo, to indicate, imply or assert compatibility or operability with the SonicDE library, must be accurate and done in good faith.

## Contributing to Sonic Branding

We appreciate your interest in contributing! To just work on the designs, download the [ZIP archive of the repository](https://github.com/Sonic-DE/sonic-branding/archive/refs/heads/main.zip) and make your changes locally. When you’re done, please create a ticket at [Issues · Sonic-DE/sonic-branding](https://github.com/Sonic-DE/sonic-branding/issues) and attach your files there.

In case you’re experienced with [Git](https://git-scm.com), you can for sure use the standard clone and pull request workflow. For Git newcomers there is also a [Git for Designers - UXpin](https://www.uxpin.com/studio/blog/git-for-designers) guide for all you need to know.
